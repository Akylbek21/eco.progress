# Активные годовые заявки

## Зачем нужна активная годовая заявка

Годовой договор с квартальным обслуживанием нельзя закрывать после выполнения первого квартала. Одна заявка остается активной весь срок договора и содержит 4 отдельных квартала с собственной работой, оплатой, документами и результатами.

Пример:

- одна заявка;
- один годовой договор;
- 4 квартала;
- по каждому кварталу отдельный счет, оплата, документы, результат и статус выполнения.

## Статус заявки и статус квартала

Общий статус заявки хранится в `Order.status`.

Для годового договора используется статус:

- `annual_active` — “Активна по годовому договору”.

Он означает, что заявка не завершена и находится на годовом обслуживании.

Статусы кварталов хранятся отдельно в `Order.quarters[].workStatus` и `Order.quarters[].paymentStatus`.

Пример:

- `Order.status = annual_active`
- `1 квартал.workStatus = completed`
- `2 квартал.workStatus = in_progress`
- `3 квартал.workStatus = planned`
- `4 квартал.workStatus = planned`

## Как создаются 4 квартала

В `Order` добавлены поля:

- `contractType`
- `contractId`
- `annualPeriodStart`
- `annualPeriodEnd`
- `quarters`

Если заявка имеет `contractType = annual_quarterly`, но в старых данных нет `quarters`, нормализация в `orderService` создает кварталы автоматически:

- из связанного finance-договора, если он найден;
- или fallback-расписанием на 4 квартала по годовому периоду.

## Документы по кварталам

Для квартала используется тип `QuarterDocument`.

Реальные бинарные файлы в `localStorage` не сохраняются. При загрузке файла frontend сохраняет только metadata:

- имя файла;
- тип файла;
- размер;
- тип документа;
- кто загрузил;
- дата загрузки.

Документы хранятся внутри конкретного квартала и не смешиваются между кварталами.

## Результаты по кварталам

Для результата используется тип `QuarterResult`.

Результат создается отдельно по кварталу и может ссылаться на документы квартала через `attachedDocumentIds`.

Тип результата определяется по этапу работ:

- лаборатория — `laboratory_protocol`;
- проектирование — `project_document`;
- вывоз — `waste_removal_act`;
- утилизация — `utilization_act`.

## Долги

Долг заявки считается как сумма `remainingAmount` всех кварталов.

Просроченный долг считается по кварталам, где:

- `remainingAmount > 0`;
- срок `dueDate` уже прошел.

В UI показываются:

- общий долг годовой заявки;
- просроченный долг;
- ближайший срок оплаты;
- предупреждения по прошлым кварталам.

## Завершение годовой заявки

Годовую заявку нельзя завершить обычным переходом после первого результата.

Завершение доступно только если:

- есть 4 квартала;
- все кварталы имеют `workStatus = completed`;
- по каждому кварталу есть результат;
- по всем кварталам нет активного долга.

После выполнения условий админ или менеджер может нажать “Завершить годовую заявку”, и статус заявки станет `Завершено`.

## Роли

Admin и manager:

- видят годовую заявку;
- управляют квартальными работами;
- загружают документы и результаты;
- могут завершить годовую заявку после выполнения условий.

Employee, ecologist, laboratory:

- ведут свои квартальные работы;
- загружают документы и результаты;
- меняют рабочий статус квартала.

Accountant:

- видит квартальные оплаты и долги;
- добавляет оплату по кварталу;
- ведет бухгалтерские комментарии.

Client:

- видит кварталы в понятном виде;
- видит статус работ и оплаты;
- загружает документы в конкретный квартал;
- видит результаты;
- не видит внутренние бухгалтерские комментарии.

## Как проверить вручную

1. Открыть раздел заявок сотрудника.
2. В фильтре “Тип заявки” выбрать “Годовые активные”.
3. Открыть `ORD-2001`.
4. Убедиться, что статус заявки — “Активна по годовому договору”.
5. Проверить, что видны 4 квартала.
6. Загрузить документ во 2 квартал и убедиться, что он появился только во 2 квартале.
7. Добавить результат по кварталу.
8. Добавить оплату по кварталу и проверить уменьшение долга.
9. Проверить, что текущий квартал подсвечивается автоматически.
10. Попробовать завершить годовую заявку до завершения всех кварталов — кнопка недоступна.
11. Завершить все 4 квартала, добавить результаты и закрыть долги.
12. Нажать “Завершить годовую заявку”.
13. Проверить, что разовые заявки работают как раньше.
