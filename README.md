# Eco.Progress

Современный frontend для рекламного сайта экологической компании Eco.Progress.

## Что реализовано

- Полностью клиентский React + Vite проект
- Tailwind CSS для адаптивного и чистого интерфейса
- React Router для маршрутизации
- React Query + Axios подготовлены для будущих API-запросов
- Аккаунт пользователя и админка построены на mock-данных
- Адаптивная верстка для desktop, tablet и mobile

## Страницы

- `/` — Главная
- `/about` — О компании
- `/services` — Список услуг
- `/services/:id` — Детали услуги
- `/employees` — Сотрудники
- `/news` — Новости
- `/news/:id` — Детали новости
- `/login` — Вход в личный кабинет
- `/cabinet` — Личный кабинет клиента
- `/admin` — Админский интерфейс (mock)

## Моковые данные

Используются локальные данные из `src/data/mockData.ts`:

- услуги
- сотрудники
- новости
- заявки
- документы
- оплаты
- уведомления
- клиенты
- профиль клиента

## Структура проекта

- `src/App.tsx` — маршрутизация и общий layout
- `src/layouts/` — макеты для публичной части, кабинета и админки
- `src/pages/` — страницы сайта
- `src/components/` — переиспользуемые UI-компоненты
- `src/services/` — сервисный уровень с mock-API
- `src/data/` — mock-данные
- `src/index.css` — стили Tailwind

## Как запустить

1. Установить зависимости:
   ```bash
   npm install
   ```
2. Запустить приложение:
   ```bash
   npm run dev
   ```
3. Открыть в браузере адрес, который покажет Vite (по умолчанию `http://localhost:4173`).

## Как подключить backend API

В проекте уже есть сервисный слой `src/services/`:

- `api.ts` — базовый Axios клиент
- `authService.ts` — mock-авторизация
- `serviceService.ts` — mock-удаленные услуги
- `newsService.ts` — mock-новости
- `orderService.ts` — mock-заявки и уведомления

При подключении backend:

1. Измените `baseURL` в `src/services/api.ts` на адрес вашего API.
2. Замените mock-реализации `getServices`, `getNewsList`, `getOrders` и пр. на реальные запросы через Axios.
3. В компонентах оставьте `useQuery` и `queryFn` — они уже готовы к работе с реальными эндпоинтами.

## Примечания

- Все данные пока используются из mock-сервиса.
- Страница входа сохраняет mock-токен в `localStorage`.
- Админка и кабинет работают полностью на frontend с локальными данными.
